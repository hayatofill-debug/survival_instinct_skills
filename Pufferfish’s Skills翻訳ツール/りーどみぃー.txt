# 📝 スキル定義・多言語翻訳自動化ツール作業マニュアル

このツールは、`puffish_skills`（MinecraftのスキルツリーMod）の定義ファイル（`definitions.json`）に含まれるテキストを自動抽出し、どの構文からであっても綺麗なトランスポートキー（オブジェクト構文）へと一括変換・パッチを当て、リソースパック用の言語ファイルを生成する開発者向けスクリプトです。

---

## 📂 構成と事前準備

作業を始める前に、ファイルが以下の配置になっているか確認してください。

    📁 (作業フォルダ)
     ├ 📄 process_skills.py    (本体スクリプト：全構文対応ハイブリッド版)
     ├ 📄 run.bat               (実行用バッチファイル：モード選択機能付き)
     └ 📁 definitions           (入力用フォルダ)
        ├ 📁 1
        │  └ 📄 definitions.json
        ├ 📁 2
        │  └ 📄 definitions.json
        └ 📁 3
           └ 📄 definitions.json

> **💡 フォルダ数について**
> フォルダ（カテゴリー）の数を増やしたり、名前を数字以外（例: `combat` や `magic`）に変更した場合は、`process_skills.py` の上部にある `TARGET_FOLDERS = ["1", "2", "3"]` を実際の構成に合わせて書き換えてください。

---

## 🔄 通常の作業フロー（3ステップ）

### 【ステップ 1】ベース言語の選択と定義ファイルのパッチ
1. `run.bat` をダブルクリックして起動します。
2. 最初にベースとなる言語モードを選択します。
   * **`[1] Japanese Base`**: 自作などの日本語が含まれる定義ファイルから抽出する場合
   * **`[2] English Base`**: 海外製のパックなど、英語が含まれる定義ファイルから抽出する場合
3. 画面の指示に従い、トランスポートキーの基準となる **識別子（例: `sis`）** を入力します。
4. メニュー画面で **`[1]`** を選択して実行します。

**✨ 自動で行われる処理と生成物:**
* `title` と `description` の中身だけをピンポイントで狙い撃ちし、元の記述が「文字列直書き」「`[{"text": "..."}]` 配列」のどちらであっても、すべて `{"translate": "識別子.shared.xxxx"}` のオブジェクト構文に自動で統一して `definitions_patched.json` を出力します。
* `rewards` や `icon`、`cost` などのシステム用データは一切書き換えずにそのまま維持します。
* 共通の出力先として `output_識別子/` フォルダが自動作成され、その中に抽出元の言語ファイル（`ja_jp.json` または `en_us.json`）と、翻訳用下地である **`blank.json`** が出力されます。

---

### 【ステップ 2】AI（Gemini等）による翻訳
1. 生成された `output_識別子/blank.json` の中身を丸ごとコピーします。
2. AI（Geminiなど）に以下のプロンプトを添えて貼り付け、翻訳を依頼します。

> **🤖 AIへの指示文テンプレート**
> 以下のJSONデータのカッコの中にあるテキストのみを、マインクラフトのスキルツリーに合う自然な英語（または対象言語）に翻訳してください。
> 
> 【注意】
> * `"TODO: Translate [...]"` の外枠の構造は、プログラムで後処理するため**絶対に消さずにそのまま残して**ください。
> * `§e` や `§c§l` などの色コード、および `\n`（改行記号）は消さずに、翻訳テキストの適切な位置に配置してください。
> * キー（`"sis.shared.xxxx"`）は変更しないでください。

3. AIから返ってきた `TODO: Translate [...]` 構造が維持されたJSONテキストを、元の `output_識別子/blank.json` に上書き保存します。

---

### 【ステップ 3】TODOの除去と最終言語ファイルの出力
1. 再び `run.bat` を起動し、ベースモードと識別子を合わせてメニューに進みます。
2. 出力したい言語に合わせて、メニューから番号を選択します。
   * **`[2]` を選択**: 定番の英語用ファイル **`en_us.json`** を一発で出力します。
   * **`[3]` を選択**: `zh_cn` や `ja_jp` など、画面で手入力した任意の言語ファイル名で出力します。

**✨ 生成されるもの:**
* `output_識別子/` フォルダの中に、`TODO` プレフィックスが綺麗に剥ぎ取られた、リソースパックにそのまま導入可能な最終言語ファイル（`en_us.json` など）が完成します。

## 🛠️ 知っておくと便利な機能

### 1. 新規スキルの追記（何度でも上書き実行OK）
すでに一度ツールにかけて `{"translate": "..."}` に置き換わっている部分は、プログラムが「処理済み」と自動判断してスキップします。後から新しいスキルを日本語や英語で追記して、再度ステップ1を実行しても、既存のキーやデータを壊すことなく新スキル分だけが `blank.json` に追記されます。

### 2. ハッシュ値の衝突リスクヘッジ（超大型プロジェクト用）
本ツールは翻訳キーを短くすっきり保つため、デフォルトで **4桁のハッシュ値（最大65,536パターン）** を使用して同じ文章を1つの共通キーに統合しています。
総テキスト数が **1,000個（約15〜20ページ相当）を超えるような超大型プロジェクト** で、万が一別の文章なのに同じキーになってしまう現象（ハッシュ衝突）を確実に防ぎたい場合は、`process_skills.py` 内の以下の箇所の数字を `[:4]` から **`[:6]`（約1,677万パターン）** に増やして実行してください。

```python
def get_text_hash(text):
    # [:4] を [:6] に増やすことで衝突リスクを完全にゼロにできます
    return hashlib.md5(text.encode("utf-8")).hexdigest()[:6]
